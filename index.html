Reflections

Vibe Coding 心得与收获

8 次作业，从产品经理到 Vibe Coder 的蜕变

什么是 Vibe Coding？

Vibe Coding 不是让你成为程序员，而是让你拥有把想法变成现实的能力。作为产品经理，我一直在写 PRD、画原型、和开发沟通。但 Vibe Coding 让我跳过了「等开发排期」这一步，直接用 AI 工具把脑子里的产品 build 出来。

这 8 次作业，就是我从「只会写文档的 PM」到「能独立交付 MVP 的产品人」的完整旅程。

01

AI 是副驾驶，不是自动驾驶

Cursor、Claude Code、Lovable 都很强，但它们需要你给出清晰的方向。需求拆解越细，Prompt 越精准，AI 输出的质量就越高。产品思维反而成了最重要的「编程技能」。

02

版本迭代是最好的老师

从链小改 v1.0 到 v5.0，我经历了数据库迁移、SEO 优化、埋点体系搭建。每次迭代都在解决真实问题，不是为了学技术而学技术。这种「需求驱动的学习」效率远超看教程。

03

Debug 是常态，不是意外

接入 LLM 时遇到 SSE 解析错误、环境变量丢失、CORS 跨域……每一个 Bug 都让我对技术架构有了更深的理解。最重要的不是写出完美代码，而是学会与 AI 一起高效排错。

04

工具链的选择决定效率

Lovable 做前端原型、Supabase 做后端和数据库、Vercel 做部署、PostHog 做埋点——这套工具组合让一个人就能完成从开发到上线的全流程。选对工具比埋头苦干重要 10 倍。

05

Skill 是 PM 的新武器

搭建小红书内容创作 Skill 让我意识到：未来的 PM 不只是写需求，还要能设计 AI 的工作流。把苏格拉底提问法和费曼学习法融入 Skill，这是产品思维和 AI 能力的真正融合。

06

从 Web 到 Native，MVP 思维不变

做了 macOS 原生应用 Scrollshot。虽然技术栈完全不同（SwiftUI vs React），但 MVP 的核心思路是一样的：找到最小可用功能，快速验证，再迭代。Vibe Coding 的本质是思维方式，不是具体技术。

07

用 AI 给 AI 作业做展示

最后一份作业是用 Claude Code 构建这个展示网站本身——从需求描述到页面上线，全程对话式完成。这正是 Vibe Coding 的终极闭环：学了什么，就用它来展示自己学了什么。

08

PM 的下一个时代已经来了

8 份作业做完，最深的感受是：会写 Prompt 的产品经理和不会的之间，差距会越拉越大。未来不存在「纯文档型 PM」，能想、能写、能 build、能部署，才是 AI 时代产品人的基本功。

最大的收获

Vibe Coding 让我从「需求的提出者」变成了「产品的构建者」。作为 AI 产品经理，我现在能独立完成从需求分析、PRD 撰写、原型搭建、LLM 接入、到部署上线的全流程。这不只是技能的提升，更是角色的转变——从等别人做，到自己做。
